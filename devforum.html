How do I properly handle JWT refresh tokens in a MERN stack app?
I'm building an authentication system and currently storing my JWT access token in localStorage. I know this is vulnerable to XSS attacks, but I can't figure out how to properly use httpOnly cookies with a React frontend and Express backend.

Here's my current setup — the token just gets stored after login:
// ❌ Current approach — vulnerable to XSS const login = async (credentials) => { const { data } = await axios.post('/api/auth/login', credentials); localStorage.setItem('token', data.accessToken); // 😬 setUser(data.user); };
What's the recommended pattern for refresh token rotation using httpOnly cookies in 2024?

✓ Accepted Answer
The correct approach is to store the refreshToken in an httpOnly cookie and keep the short-lived access token in memory (React state). Here's the pattern:
// ✅ Express: set refresh token as httpOnly cookie res.cookie('refreshToken', refreshToken, { httpOnly: true, secure: process.env.NODE_ENV === 'production', sameSite: 'strict', maxAge: 7 * 24 * 60 * 60 * 1000 // 7 days }); res.json({ accessToken }); // short-lived, keep in memory
On the React side, store the access token in a ref or state — never in localStorage. Use an Axios interceptor to refresh it silently when it expires.
